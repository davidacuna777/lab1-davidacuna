- Diseño mobile-first, responsive, accesible (`aria-label`, `focus-visible`).  
- Navbar fijo con menú hamburguesa y scroll con offset.  
- Sin dependencias externas salvo Google Fonts.

### Personaliza antes de publicar

1. Define una acción real para el formulario o deja claro que es demostrativo.  
2. Agrega tu enlace final de Figma en la sección de entregables.  
3. Mantén actualizada la sección de proyectos con evidencia nueva.  
4. Revisa que todas las secciones reflejen tu experiencia actual.

### Despliegue en GitHub Pages

1. `git add . && git commit -m "feat: primera versión del portafolio"`.  
2. Sube a GitHub y en Settings → Pages elige la rama (`main`) y carpeta `/root`.  
3. Espera el build y pega el enlace resultante en este README.  
4. Valida manualmente el menú móvil, los enlaces externos y Lighthouse (Performance/SEO/Accessibility).

## 🎯 Perfil Profesional Objetivo

A partir del análisis de estas 14 ofertas laborales reales, el perfil profesional objetivo definido es:

> **Desarrollador Web Front-End con capacidades Full Stack**, enfocado en la construcción de interfaces modernas, accesibles y responsive utilizando HTML5, CSS3, JavaScript y React/Next.js, con conocimientos de back-end básico (Node.js o Python) y capacidad de colaborar en equipos ágiles de forma remota.

En la versión del portafolio desarrollada para esta entrega, el posicionamiento profesional se presenta como un perfil con experiencia en proyectos académicos y freelance, mostrando alcance en diseño UI, desarrollo front-end y bases de integración back-end, sin salir de las tecnologías permitidas por el curso (HTML, CSS y JavaScript puro).

### Justificación del perfil

La mayoría de las ofertas de nivel junior a mid-level exigen un sólido dominio de HTML/CSS/JavaScript junto con al menos un framework moderno (React es el más frecuente). El perfil de portafolio web desarrollado en esta actividad responde directamente a lo que buscan los reclutadores: demostrar habilidades técnicas reales, proyectos propios y presentación profesional.

El sitio web de portafolio estará orientado a **postulación a un puesto ficticio de Junior Frontend Developer**, mostrando proyectos, habilidades técnicas y contacto, construido únicamente con HTML5, CSS3 y JavaScript puro (sin frameworks), aplicando los mismos estándares de calidad que exigen las ofertas investigadas.

### 🌟 Diferenciadores clave

1. **Énfasis en accesibilidad web (WCAG 2.1):** Pocas ofertas junior la exigen explícitamente, pero sí la valoran. Implementar prácticas de accesibilidad desde el portafolio demuestra madurez técnica inusual para un perfil junior, representando una ventaja competitiva real.

2. **Pipeline completo Figma → Código:** El proceso de diseñar primero en Figma y luego implementarlo pixel-perfect en HTML/CSS demuestra la capacidad de trabajar con equipos de diseño, una habilidad muy demandada según 7 de 14 ofertas analizadas.
